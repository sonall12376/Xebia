Phase 1: Initial Setup & Database Schemas

The initialization phase established a clean, modular mono-repository directory system, set up connection clients to MongoDB, configured PostCSS with Tailwind CSS v4 directives, and structured Mongoose database schemas with rigorous validations.

Database Schema Implementations

1. User Schema (backend/models/User.js)

  • name: String (required).
  • email: String (required, unique, regex validation).
  • password: Hashed using bcryptjs with auto salt generation via pre-save Mongoose hook.
  • role: Enum of ['Admin', 'Manager', 'Employee'] (default: 'Employee').
  • department: String.
  • designation: String.
  • contact: String.
  • joiningDate: Date (default: Date.now).
  • managerId: Relational self-reference to User table to manage reporting hierarchies.

2. Project Schema (backend/models/Project.js)

  • name: String (required).
  • description: String.
  • startDate / endDate: Timeline Dates.
  • status: Enum of ['Not Started', 'In Progress', 'Completed', 'On Hold'] (default: 'Not Started').
  • priority: Enum of ['Low', 'Medium', 'High'] (default: 'Medium').
  • managerId: Reference pointing to evaluating Project Manager (User).
  • employeeIds: Array of references pointing to assigned team members (User).
  • isDeleted: Boolean (default: false) for soft deletions.

3. Attendance Schema (backend/models/Attendance.js)

  • userId: Reference linking the log to the Employee (User, required).
  • date: Date (normalized to midnight UTC to prevent time-zone shifts).
  • checkInTime: ISO Date timestamp (required).
  • checkOutTime: ISO Date timestamp (optional).
  • Compound Index: Unique index on { userId: 1, date: 1 } to prevent double check-ins.

4. Review Schema (backend/models/Review.js)

  • employeeId: Reference to evaluated Employee (User, required).
  • managerId: Reference to submitting Evaluator (User, required).
  • 5-Star rating fields: Integers between 1 and 5 (Technical, Communication, Team Collaboration, Problem Solving, Leadership).
  • comments: String text comments.

Module 1: Authentication & Authorization

Module 1 built secure user onboarding views and API routes, protected endpoints with signed JWT sessions, and blocked invalid navigation using client-side routing guards.

Features Implemented

  • JWT Session Generation: Cryptographically signs user IDs and roles into a token payload.
  • Endpoint Protection Middleware: Protects routes by validating Bearer tokens.
  • Role-based Authorization: Validates roles before granting access to server endpoints.
  • Client Router Guards: Wraps routes in a ProtectedRoute component to reject invalid sessions.
  • Onboarding UI: Login, Signup, and ForgotPassword screens.

API Endpoint Registry

Endpoint Method Access Description
/api/auth/register POST Public Creates a user, hashes password, and returns profile + signed JWT.
/api/auth/login POST Public Validates credentials, returning profile + signed JWT.
/api/auth/me GET Private Returns active user profile. Requires Bearer Token.

Module 2: Role-based Dashboards

Module 2 created responsive dashboard overview screens tailored specifically to the logged-in user's role.

Features Implemented

  • Admin Console: Displays employee counts, active projects, live attendance metrics, review progress counts, and system activity streams.
  • Manager Console: Displays team directories with active status badges, managed project tables, and daily reports checklist.
  • Employee Console: Displays personal shifts status, overall score averages, score breakdown charts, and manager reviews feeds.

API Endpoint Registry

Endpoint Method Access Description
/api/dashboard/admin GET Private (Admin) Fetches total employees, active projects, today's attendance %, and logs feed.
/api/dashboard/manager GET Private (Manager) Fetches team size, assigned projects, check-in logs, and pending report reviews.
/api/dashboard/employee GET Private (Employee) Fetches assigned projects, today's shifts, average rating scores, and review comments.

Module 3: Employee CRUD Management (Admin Only)

Module 3 implemented directory boards, CRUD forms, and cascading deletions to manage system user directories.

Features Implemented

  • Employee Directory Board: Dynamic search filters by name/email, designations, departments, and roles.
  • CRUD Forms: Enforces strict data validations (unique emails, valid contact formats) and conditional password checks.
  • Dynamic Manager Dropdown: Pulls current user list with role Manager. Hides selection if user role is Manager.
  • Cascading Deletions: Deletes user references in assigned projects and direct reports before user profile removal.

API Endpoint Registry

Endpoint Method Access Description
/api/users GET Private (Admin) Returns a list of all user profiles (excluding Admins).
/api/users POST Private (Admin) Creates and registers a new employee/manager profile.
/api/users/:id PUT Private (Admin) Updates user parameters. Password update is optional.
/api/users/:id DELETE Private (Admin) Removes profile and detaches reporting/project assignments.
/api/users/managers GET Private (Admin) Fetches dropdown managers list.

Module 4: Project Management & Soft Deletions

Module 4 implemented project tracking, status lanes, multi-select employee assignments, and database soft-deletes.

Features Implemented

  • Project Management Board: Displays status lanes, priorities, timelines, and approaching deadline indicators.
  • Employee Checklist: Renders a checklist of employees to check/uncheck multiple assignees in forms.
  • Soft Delete Schema: Sets isDeleted = true to filter out projects while preserving historic database relations.
  • Role Boundaries: Managers can only edit/delete projects they own; Admins have full access.

API Endpoint Registry

Endpoint Method Access Description
/api/projects GET Private Returns list of non-deleted projects based on user roles.
/api/projects POST Private (Admin/Manager) Registers a new project. Default manager is req.user for Managers.
/api/projects/:id PUT Private (Admin/Manager) Updates project details. Blocks unauthorized managers.
/api/projects/:id DELETE Private (Admin/Manager) Sets project isDeleted = true. Blocks unauthorized managers.

Module 5: Attendance Management & Live Shifts

Module 5 implemented clock-in/out widgets, decimal work hour calculations, active session indicators, and role history auditing with manager self-attendance tab options.

Features Implemented

  • Clock Widget: Dashboard buttons triggering clock-in and clock-out timestamps. Blocks double check-ins.
  • Hours Calculator: Computes work hours as checkOutTime - checkInTime in decimal. Active shifts show "In Progress" with hours calculated to current clock time.
  • Tabbed Layout (Manager/Admin): Toggle options to view Team Presence Audit (direct reports logs) and My Attendance (personal log metrics).

API Endpoint Registry

Endpoint Method Access Description
/api/attendance/check-in POST Private Registers check-in for today. Fails if record exists.
/api/attendance/check-out POST Private Updates today's check-out timestamp.
/api/attendance/status GET Private Returns today's check-in status log (or null).
/api/attendance/my-history GET Private Returns personal logs sorted by date descending.
/api/attendance/team GET Private (Admin/Manager) Returns direct reports' attendance (Managers) or all logs (Admin).

Module 6: Performance Review Management

Module 6 implemented performance evaluation logs with 5-star ratings and textual feedback comments, completed by direct managers, and auditable by HR admins.

Features Implemented

  • Star Rating Inputs: Custom, interactive 5-star controls to evaluate Technical, Communication, Teamwork, Problem Solving, and Leadership skills.
  • Manager Evaluation Board: Displays reports waiting evaluations (Pending tab) and logs of submitted reviews with update/delete actions.
  • Organizational Audit Logs (Admin): HR console listing all submitted reviews with employee/evaluator search filters.
  • Feedback History (Employee): Log history displaying evaluations and comments from reporting managers.

API Endpoint Registry

Endpoint Method Access Description
/api/reviews POST Private (Admin/Manager) Submits a new review. Blocks duplicate evaluations.
/api/reviews/my-reviews GET Private Returns all evaluations received by the user.
/api/reviews/manager GET Private (Manager) Returns all evaluations submitted by this manager.
/api/reviews/:id PUT Private (Admin/Manager) Updates review ratings/comments. Blocks unauthorized managers.
/api/reviews/:id DELETE Private (Admin/Manager) Deletes evaluation. Blocks unauthorized managers.

System Installation & Startup Guide

Follow these setup guidelines to run and verify the local environment:

1. Setup Prerequisites

Ensure you have Node.js and a local instance of MongoDB (default port 27017) running.

2. Bootstrap the Backend Server

cd backend
npm install
node scripts/seed.js     # Seeds initial users, projects, and logs
npm run dev              # Runs Express server on http://localhost:5000

3. Start the Frontend Client

cd ../frontend
npm install
npm run dev              # Runs Vite Client on http://localhost:5173

4. Execute Verification Test Suites

Run any of the automated API verification scripts in the backend directory to test functionality:

  • node scripts/verify-auth.js (Authentication)
  • node scripts/verify-schemas.js (Database validations)
  • node scripts/verify-crud.js (Admin Employee actions)
  • node scripts/verify-projects.js (Project Management soft-deletes)
  • node scripts/verify-attendance.js (Check-ins/outs)
  • node scripts/verify-reviews.js (Evaluations)
